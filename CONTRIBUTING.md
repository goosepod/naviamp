# Contributing

Naviamp is private while the project is forming, but the repository should stay ready for a future open-source release.

## Project Style

- Keep provider-specific code behind provider modules.
- Keep UI code out of the domain layer.
- Add tests for domain logic and provider mapping.
- Prefer small, readable changes.
- Document decisions that affect architecture or future contributors.

## Sharing Changes Back

The Apache-2.0 license is permissive and does not require forks to publish their changes. The project should still make upstream contribution easy:

- keep setup documented
- keep tests runnable with one command
- keep issues and pull requests focused
- explain design choices in docs when they are not obvious
